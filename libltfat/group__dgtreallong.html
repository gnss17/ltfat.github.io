Initialize plan for Discrete Gabor Transform for real signals for the factorization algorithm

Note
f can be NULL if the plan is intended to be used with the _newarray execute function. Similarly, c can also be NULL, but only if FFTW_ESTIMATE is passed in flags (the FFTW planning routine does not touch the array). On the other hand, the content of c might get overwritten if other FFTW planning flags are used.
Parameters
[in]fInput signal, size L x W or NULL
[in]gWindow, size L x 1
[in]LSignal length
[in]WNumber of channels of the signal
[in]aTime hop factor
[in]MNumber of frequency channels
[in]cDGT coefficients, size M2 x N x W or NULL if (flags & FFTW_ESTIMATE) is nonzero.
[in]ptypePhase convention
[in]flagsFFTW plan flags
[out]planDGT plan
